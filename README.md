# Asana リマインダー（制作・更新プロジェクト用）

制作・更新プロジェクトを **10分ごとに自動チェック**し、対応もれを防ぐための Windows 常駐ツールです。
条件に合うタスクがあると、画面右上に**点滅する警告ポップアップ**を表示します（音は鳴りません）。
放置しても10分ごとに最新情報へ更新されます。

Python 標準機能のみで動作し、追加ライブラリのインストールは不要です。

> **設定は2ファイル構成**：共通設定（キーワード・通知時間など）は `settings.ini`（Gitで全員に配布）、各自のトークンは `config.ini`（Git管理外）に分けています。共通設定を変えたら **git push → 各自が git pull → 再起動** だけで全員に反映されます。

---

## 通知の種類

### ① チーム内確認待ちアラート（赤・点滅）
- Status＝「チーム内確認待ち」が **しきい値（既定2件）以上**でポップアップ。
- **17時以降**は1件でも通知し、×で閉じても **0件になるまで**出し続けます（退勤前の取りこぼし防止）。

### ② 今日の時間指定タスクアラート（オレンジ→赤・点滅）
- **今日期日** × タスク名に指定キーワード（AM / 17時 / 10:00 / ASAP / 流し込み / マスト など）を含むタスクが1件以上でポップアップ。
- Status が「作業完了」になるまで、閉じても通知し続けます。
- 「流し込み」「表示確認」は**担当者が割り当てられたら通知しません**（担当が対応するため）。

> 通知は **平日9時〜22時**のみ。夜間（22時〜翌9時）と土日は通知しません。

## セットアップ（各自1回だけ）

> **前提：Python が必要です。** 入っていない、または動作テストでエラーが出る場合は **BPR部** にご相談ください（ソフト導入はBPR部の担当です）。

1. **クローン（ダウンロード）**
   このリポジトリを自分のPCにクローンします。ZIPで取得した場合は、ZIPを右クリック→プロパティ→「ブロックの解除」にチェック→OK してから展開してください。

2. **Asanaトークンを取得**
   [https://app.asana.com/0/my-apps](https://app.asana.com/0/my-apps) を開く →「新しいアクセストークンを作成」→ 名前を付けて作成 → 表示された文字列をコピー。
   ⚠ トークンは自分専用です。他人に共有・GitHubにアップしないでください。

3. **初回起動してトークンを貼る**
   `動作テスト.bat` をダブルクリック → 初回は `config.ini` が自動生成され、メモ帳で開きます。
   `token = ここに自分のトークンを貼る` の右側を、コピーしたトークンに置き換えて上書き保存（Ctrl+S）。

4. **動作確認**
   もう一度 `動作テスト.bat` を起動 → 黒い画面が出てエラーが無ければOK。

5. **自動起動を登録**
   `PC起動時に自動起動する_登録.bat` をダブルクリック →「OK: registered」と出れば完了。次回ログインから自動で動きます。
   今すぐ動かすなら `常駐起動.vbs` をダブルクリック（画面には何も出ませんが裏で動きます）。

## ファイル構成

| ファイル | 説明 |
|---|---|
| `asana_reminder.py` | 本体プログラム（触らない） |
| `settings.ini` | **共通設定**（全員共通・Gitで配布・管理者が編集） |
| `config.ini` | 各自の**トークンだけ**を入れる（**Git管理外・自動生成**） |
| `config.sample.ini` | トークン設定の見本（`config.ini` のもと） |
| `常駐起動.vbs` | 裏で起動する（普段使い） |
| `動作テスト.bat` | 動作確認用（黒い画面でログが見える） |
| `PC起動時に自動起動する_登録.bat` | 自動起動の登録 |
| `自動起動を解除する.bat` | 自動起動の解除 |
| `はじめにお読みください（使い方）.txt` | 詳しい使い方・FAQ |
| `asana_reminder.log` | 動作ログ（**自動生成・Git管理外**） |

> 読み込みは `settings.ini` を優先し、無いキーだけ `config.ini` で補います（旧バージョン互換）。トークンは `config.ini` からのみ読み、`settings.ini` からは絶対に読みません。

## 設定変更（共通設定 = `settings.ini`）

キーワード・しきい値・通知時間などの共通設定は `settings.ini`（各項目の意味は同ファイルのコメント参照）を編集します。**変更して git push → 各自が git pull → 再起動** すれば全員に反映されます。

| 項目 | 意味 | 既定 |
|---|---|---|
| `threshold` | ①確認待ちの通知しきい値（件） | 2 |
| `evening_hour` / `evening_threshold` | 夕方ルール（何時以降・何件で通知） | 17時 / 1件 |
| `active_start_hour` / `active_end_hour` | 通知する時間帯 | 9〜22時 |
| `quiet_weekend` | 土日は通知しない | true |
| `interval_minutes` | チェック間隔（分） | 10 |
| `[today_alert] keywords` | ②の検知キーワード | AM,PM,午前,午後,時まで,ASAP,至急,マスト,流し込み,表示確認,設定確認 |
| `[today_alert] assignee_optional_keywords` | 担当者が入っていれば通知しないキーワード | 流し込み,表示確認,設定確認 |

## 更新のしかた（すでにインストール済みの人）

共通設定が変わったら、各自は以下だけでOK（`config.ini` のトークンは触らない）：

1. ツールのフォルダで最新を取得：`git pull`
2. リマインダーを再起動（タスクマネージャーで `pythonw.exe` を終了 → `常駐起動.vbs`。またはPCを再起動）

---

## 止めたいとき

- 自動起動をやめる：`自動起動を解除する.bat` を実行。
- 今すぐ止める：タスクマネージャーで `pythonw.exe` を終了。

---

## セキュリティ

- 個人アクセストークンは `config.ini` にのみ保存され、`.gitignore` で **Git管理外**です。GitHubに上がることはありません。
- ログにトークンは記録しません。
- トークンは**自分専用**です。絶対に他人に共有・公開しないでください。
